# Release Notes Template

Fill from the PR honestly. Keep it tight. If a section has nothing, write "None".

## Summary
One or two sentences: what this release does and why it matters to a user.

## Changes
- Bullet each user-visible change, grounded in the diff (not the description's claims).

## Breaking
- List any signature/behaviour change that affects existing callers. If none, "None".

## Migration steps
- Concrete, ordered steps an operator runs to adopt this release (schema migrations,
  config/env changes, data backfills, version bump). If none, "None".
